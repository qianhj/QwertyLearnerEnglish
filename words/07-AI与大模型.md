# 07 AI & 大模型

> LLM、训练推理、Prompt 工程、AI 安全、向量检索相关词汇

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ accuracy | 准确率 | 分类正确的样本占总样本的比例 |
| ⭐ adversarial | 对抗性的 | `adversarial prompt` 试图绕过模型安全限制的输入 |
| ⭐ agent | AI 智能体 | 能自主调用工具、执行多步骤任务的 LLM 应用 |
| ⭐ alignment | 对齐 | 使模型行为符合人类价值观和意图的研究方向 |
| api-key | API 密钥 | `Authorization: Bearer sk-...` 调用 AI API 的身份凭证 |
| ⭐ assistant | 助手角色 | `role: "assistant"` 对话中模型扮演的角色 |
| ⭐ attention | 注意力机制 | Transformer 核心机制，`self-attention` 计算词间关联 |
| ⭐ augmented | 增强的 | `Retrieval-Augmented Generation (RAG)` 检索增强生成 |
| ⭐ benchmark | 基准测试 | `MMLU / HumanEval` 评估模型能力的标准测试集 |
| ⭐ chain-of-thought | 思维链 | `Chain-of-Thought prompting` 让模型逐步推理再给答案 |
| ⭐ checkpoint | 检查点 | 训练过程中保存的模型权重快照，用于恢复训练 |
| ⭐ classification | 分类任务 | 将输入分到预定类别，如情感分析（正面/负面） |
| ⭐ clustering | 聚类 | 无监督学习，将相似数据分组 |
| ⭐ completion | 文本补全 | `chat completion API` 模型根据输入生成后续文本 |
| ⭐ context-window | 上下文窗口 | 模型单次能处理的最大 token 数，如 128k tokens |
| ⭐ cosine | 余弦相似度 | `cosine similarity` 计算两个向量的相似程度（0~1）|
| ⭐ cuda | CUDA | NVIDIA GPU 并行计算框架，AI 训练的底层加速 |
| ⭐ dataset | 数据集 | 用于训练或评估模型的标注数据集合 |
| ⭐ diffusion | 扩散模型 | `diffusion model` Stable Diffusion 等图像生成模型的基础 |
| ⭐ distillation | 知识蒸馏 | 用大模型（教师）指导小模型（学生）训练，压缩模型 |
| ⭐ embedding | 向量嵌入 | 将文本/图像转为高维向量，用于语义检索 |
| ⭐ epoch | 训练轮次 | 模型完整遍历一次训练集称为一个 epoch |
| ⭐ evals | 评估测试 | `model evals` 测量模型性能和安全性的评估套件 |
| ⭐ few-shot | 少样本学习 | `few-shot prompting` 在提示中给出少量示例引导输出 |
| ⭐ fine-tune | 微调 | 在预训练模型基础上用特定数据集继续训练 |
| ⭐ function-calling | 函数调用 | LLM 输出结构化 JSON 来调用外部工具/API |
| ⭐ gradient | 梯度 | 反向传播中计算的损失函数偏导，用于更新权重 |
| ⭐ grounding | 基础事实锚定 | 让模型输出基于真实数据，减少幻觉 |
| ⭐ guardrail | 护栏 | 防止模型输出有害内容的安全过滤机制 |
| ⭐ hallucination | 幻觉 | 模型生成看似合理但实际错误的内容 |
| ⭐ inference | 推理（运行） | 用训练好的模型处理新输入并生成输出 |
| ⭐ jailbreak | 越狱 | 试图绕过模型安全限制获取被禁止输出的攻击方式 |
| ⭐ label | 标签 | 数据集中人工标注的正确答案 |
| latent | 潜在空间 | `latent space` 模型内部的高维特征表示空间 |
| ⭐ layer | 层 | 神经网络中的处理单元，深度学习靠堆叠多层实现 |
| ⭐ leaderboard | 排行榜 | `LMSYS Chatbot Arena` 比较不同模型性能的公开榜单 |
| ⭐ lora | LoRA 微调 | Low-Rank Adaptation，低参数量高效微调方法 |
| ⭐ loss | 损失函数 | 衡量模型预测与真实标签差距的函数，训练目标是最小化 loss |
| ⭐ model | 模型 | 经过训练的神经网络，如 `claude-3-5-sonnet` |
| ⭐ multimodal | 多模态 | 同时处理文本、图像、音频等多种输入类型的模型 |
| ⭐ neural network | 神经网络 | 模仿人脑神经元连接的机器学习模型结构 |
| open-source | 开源模型 | `Llama / Mistral` 权重公开的模型，可本地部署 |
| ⭐ precision | 精确率 | 预测为正例中真正为正例的比例 |
| ⭐ pretrain | 预训练 | 在大规模数据上训练基础模型，之后再微调 |
| ⭐ prompt | 提示词 | 发送给模型的输入文本，决定模型行为 |
| ⭐ pruning | 剪枝 | 删除模型中不重要的权重，减小模型体积 |
| ⭐ quantization | 量化 | 将模型权重从 float32 压缩为 int8/4bit，减少内存占用 |
| ⭐ rag | 检索增强生成 | Retrieval-Augmented Generation，搜索知识库后再生成 |
| ⭐ recall | 召回率 | 实际为正例中被正确预测的比例 |
| ⭐ red-teaming | 红队测试 | 模拟攻击者测试模型安全漏洞 |
| ⭐ retrieval | 检索 | 从知识库中找到相关文档，RAG 的第一步 |
| ⭐ rlhf | 人类反馈强化学习 | Reinforcement Learning from Human Feedback，对齐训练方法 |
| ⭐ safety | 安全性 | 模型不输出有害内容的能力 |
| ⭐ sampling | 采样 | 生成文本时从概率分布中选取下一个 token 的策略 |
| ⭐ serving | 模型服务化 | 将训练好的模型部署为可调用 API |
| ⭐ similarity | 相似度 | `cosine similarity` 向量间的语义接近程度 |
| system-prompt | 系统提示词 | `role: "system"` 设定模型角色和行为规范的提示 |
| ⭐ temperature | 温度参数 | 控制输出随机性，0=确定性，1=创造性，>1=随机 |
| ⭐ token | 词元 | 模型处理文本的最小单位，约 4 个字符或 0.75 个英文词 |
| tokens-per-second | 每秒生成 token 数 | 衡量推理速度的指标，`TPS` |
| ⭐ tool-use | 工具调用 | 模型调用外部工具（搜索、代码执行）的能力 |
| ⭐ top-k | Top-K 采样 | 只从概率最高的 K 个 token 中采样 |
| ⭐ top-p | Top-P 采样 | 从累积概率达到 P 的 token 集合中采样（nucleus sampling）|
| ⭐ training | 训练 | 用数据反复调整模型权重使 loss 下降的过程 |
| ⭐ transformer | Transformer 架构 | 现代 LLM 的基础架构，基于注意力机制 |
| ⭐ vector | 向量 | 语义嵌入后的数值数组，用于相似度计算 |
| ⭐ vram | 显存 | GPU 的内存，限制可加载的模型大小 |
| ⭐ weight | 模型权重 | 神经网络中经过训练调整的参数值 |
| ⭐ zero-shot | 零样本学习 | `zero-shot prompting` 不给示例，直接让模型完成任务 |

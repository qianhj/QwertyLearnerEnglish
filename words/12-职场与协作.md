# 12 职场与协作

> 代码审查、技术沟通、日常协作必备词汇

## 代码审查 (Code Review)

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ LGTM | Looks Good To Me | `LGTM 👍` 审批通过 |
| ⭐ nit / nitpick | 吹毛求疵的小问题 | `nit: rename this variable` 小问题：变量名改一下 |
| ⭐ blocking | 阻塞性问题（必须改）| `this is blocking` 这必须改才能合 |
| ⭐ non-blocking | 非阻塞问题（建议）| `non-blocking suggestion` 建议性意见 |
| ⭐ suggestion | 建议（GitHub功能）| `left a suggestion` 留了个修改建议 |
| ⭐ approve | 批准；同意 | `request changes` vs `approve` |
| ⭐ request changes | 请求修改 | PR状态：需要修改 |
| ⭐ PTAL | Please Take A Look | `PTAL @someone` 请看一下 |
| ⭐ CC | Carbon Copy | `CC @manager` 抄送某人 |
| workaround | 临时解决方案 | `temporary workaround` 临时绕过方案 |
| follow-up | 后续跟进 | `address in follow-up` 后续再处理 |
| out of scope | 超出范围 | `this is out of scope` 超出本次范围 |
| edge case | 边界情况 | `handle this edge case` 处理这个边界情况 |
| corner case | 极端情况 | `rare corner case` 罕见的极端场景 |
| sanity check | 合理性检查 | `do a sanity check` 检查是否合理 |

|---|---|---|

| ⭐ privilege | 权限 | `privilege escalation` 权限提升 |

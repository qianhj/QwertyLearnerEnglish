# 09 技术社区（GitHub / Stack Overflow / HN）

> 技术社区互动常用词汇，包括代码评审黑话、论坛术语和开发者惯用表达

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ ACK | 认可；同意（代码评审） | `ACK, merging` 同意，合并了 |
| ⭐ anti-pattern | 反模式；不良实践 | `this is an anti-pattern` 这是反模式 |
| ⭐ Ask HN | 向 HN 社区提问 | `Ask HN: How do you...` 向 HN 提问 |
| badge | 徽章（Stack Overflow） | `gold badge` 金牌徽章 |
| bike-shedding | 在细枝末节上争论 | `stop bike-shedding the color` 别纠结颜色了 |
| bounty | 悬赏（Stack Overflow） | `offer a bounty` 挂悬赏求答案 |
| ⭐ bug report | 缺陷报告 | `file a bug report` 提交 Bug 报告 |
| bus factor | 巴士系数（人员风险） | `bus factor of 1` 只有一人了解该模块，风险极高 |
| cargo cult | 照抄代码（不求甚解） | `cargo cult programming` 不理解就复制粘贴 |
| ⭐ close vote | 关闭投票（SO） | `vote to close as duplicate` 投票标记为重复问题 |
| ⭐ contributor | 贡献者 | `top contributor` 核心贡献者 |
| ⭐ dead | 已删除/屏蔽内容（HN） | `[dead] comment` 被管理员删除的评论 |
| ⭐ DNM | 不要合并（Do Not Merge） | `DNM: still WIP` 不要合并，还在开发中 |
| dogfooding | 自己用自己的产品 | `we're dogfooding the new API` 我们在自测新 API |
| ⭐ downvote | 踩；负面投票 | `got downvoted` 被踩了 |
| ⭐ DRY | 不重复原则（Don't Repeat Yourself） | `violates DRY` 违反了 DRY 原则 |
| ⭐ AFAIK | 据我所知（As Far As I Know） | `AFAIK it's a known bug` 据我所知这是已知 bug |
| ⭐ FYI | 仅供参考（For Your Information） | `FYI: the API changed` 仅供参考：API 改了 |
| ⭐ IIRC | 如果我没记错（If I Recall Correctly） | `IIRC it was fixed in v2` 如果我没记错是 v2 修的 |
| ⭐ IMO / IMHO | 在我看来（In My (Humble) Opinion） | `IMO this approach is cleaner` 我觉得这个方案更清晰 |
| karma | 积分；声望（HN/Reddit） | `need 501 karma to downvote` 需要 501 积分才能踩 |
| ⭐ KISS | 保持简单原则（Keep It Simple） | `KISS principle` 别过度设计 |
| ⭐ LGTM | 看起来不错（Looks Good To Me） | `LGTM, approved` 没问题，批准 |
| low-hanging fruit | 容易实现的任务 | `tackle low-hanging fruit first` 先做简单的任务 |
| ⭐ maintainer | 维护者；项目负责人 | `ask a maintainer` 联系项目维护者 |
| ⭐ MVP | 最小可行产品（Minimum Viable Product） | `ship an MVP first` 先发布最小可行版本 |
| NACK / NAK | 拒绝；不同意（代码评审） | `NACK, this breaks the API` 不同意，这会破坏 API |
| ⭐ post-mortem | 故障复盘；事后分析 | `write a post-mortem` 写故障复盘报告 |
| ⭐ PTAL | 请帮忙看一下（Please Take A Look） | `PTAL at this PR` 请看一下这个 PR |
| rabbit hole | 越挖越深的问题 | `fell into a rabbit hole` 越研究越深陷其中 |
| ⭐ reputation | 声望值（Stack Overflow） | `gain reputation` 提升声望值 |
| ⭐ RFC | 征求意见稿（Request For Comments） | `RFC: new config format` 提议讨论新配置格式 |
| rubber duck debugging | 橡皮鸭调试法 | `rubber duck debugging` 对着橡皮鸭讲解来找 bug |
| ⭐ SGTM | 听起来不错（Sounds Good To Me） | `SGTM, let's go` 没问题，开搞吧 |
| ⭐ Show HN | 向 HN 社区展示作品 | `Show HN: I built X` 向 HN 展示我做的东西 |
| ⭐ sprint | 迭代冲刺（Agile） | `two-week sprint` 两周一个迭代 |
| ⭐ TBD | 待定（To Be Determined） | `timeline TBD` 时间待定 |
| ⭐ tech debt | 技术债 | `pay down tech debt` 还技术债 |
| ⭐ TL;DR | 太长不看；摘要 | `TL;DR: use option B` 简而言之：用方案 B |
| ⭐ upvote | 点赞；正面投票 | `most upvoted answer` 最高赞的回答 |
| ⭐ WIP | 进行中（Work In Progress） | `WIP: not ready for review` 开发中，暂不需要 review |

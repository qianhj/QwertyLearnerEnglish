# 06 云服务 & DevOps

> AWS/Azure/GCP、Docker、Kubernetes、CI/CD、网络、监控、安全合规相关词汇

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ acl | 访问控制列表 | `network ACL` 子网级别的网络访问规则 |
| ⭐ alert | 告警 | 超过阈值时触发通知，叫醒 on-call 工程师 |
| ⭐ artifact | 构建产物 | `build artifact` CI 输出的可部署文件 |
| ⭐ autoscaling | 自动扩缩容 | `auto-scaling group` 按负载自动增减实例数 |
| availability zone | 可用区 | `AZ` 同区域内的独立数据中心，用于高可用 |
| ⭐ blue-green | 蓝绿部署 | 两套环境切换流量，实现零停机发布 |
| ⭐ bucket | 存储桶 | `S3 bucket` 存储静态文件、备份、日志 |
| ⭐ canary | 金丝雀发布 | `canary deployment` 先对 1% 流量发布，验证后逐步放量 |
| ⭐ cdn | 内容分发网络 | `CDN cache` 将静态资源缓存到边缘节点，加速访问 |
| ⭐ ci/cd | 持续集成/持续交付 | `GitHub Actions CI/CD pipeline` 自动化构建、测试、部署 |
| ⭐ cidr | 无类别域间路由 | `10.0.0.0/24` IP 地址段的表示方法 |
| ⭐ cloudwatch | 云监控（AWS） | `CloudWatch metrics/logs/alarms` AWS 监控服务 |
| ⭐ cluster | 集群 | `Kubernetes cluster` 多节点协同工作的系统 |
| compliance | 合规 | `SOC2 / GDPR compliance` 满足监管和安全要求 |
| ⭐ configmap | 配置映射（K8s） | `K8s ConfigMap` 存储非敏感配置数据 |
| ⭐ container | 容器 | `Docker container` 轻量化隔离的运行环境 |
| ⭐ daemonset | 守护进程集（K8s） | 每个节点上自动运行一个 Pod，用于日志收集等 |
| ⭐ deployment | 部署控制器（K8s） | `kubectl apply -f deployment.yaml` 声明式管理 Pod |
| ⭐ dns | 域名解析系统 | `DNS record` A/CNAME 记录，域名→IP |
| ⭐ docker | Docker 容器工具 | `docker build` / `docker run` / `docker push` |
| ⭐ dockerfile | Docker 构建文件 | 描述镜像构建步骤，`FROM node:20 ... RUN npm install` |
| ⭐ ec2 | 云服务器（AWS） | `EC2 instance` 弹性计算实例，按需付费 |
| ⭐ environment | 运行环境 | `dev / staging / production` 三套环境 |
| ⭐ feature flag | 功能开关 | 在生产中动态开启/关闭某功能，无需重新发布 |
| ⭐ firewall | 防火墙 | `security group` AWS 实例级别的防火墙规则 |
| gdpr | 通用数据保护条例 | 欧盟个人数据保护法规，违规罚款极高 |
| ⭐ helm | K8s 包管理工具 | `helm install chart` 一键部署复杂 K8s 应用 |
| ⭐ hpa | 水平自动扩缩容（K8s） | `HorizontalPodAutoscaler` 按 CPU/内存自动扩缩 Pod 数 |
| ⭐ iam | 身份访问管理（AWS） | `IAM role / policy` 控制谁能访问哪些 AWS 资源 |
| ⭐ image | 容器镜像 | `docker pull node:20` 镜像是容器的模板 |
| ⭐ incident | 生产事故 | `P1 incident` 影响用户的高级别故障 |
| ⭐ ingress | 入口规则（K8s） | `Ingress controller` 管理外部流量进入集群 |
| ⭐ instance | 云主机实例 | `EC2 instance` / `RDS instance` |
| job | CI 任务 | CI pipeline 中的单个执行单元 |
| ⭐ kubernetes / k8s | 容器编排系统 | 自动管理容器的部署、调度、扩缩容、自愈 |
| ⭐ lambda | 函数计算（AWS） | `Lambda function` 事件驱动的无服务器函数 |
| ⭐ load balancer | 负载均衡器 | `ALB / NLB` 将流量分发到多个后端实例 |
| ⭐ log | 日志 | `CloudWatch Logs` / `kubectl logs pod-name` |
| ⭐ manifest | 配置清单（K8s） | `K8s manifest YAML` 声明资源期望状态的配置文件 |
| ⭐ metric | 指标 | `CPU metric` / `request count` 可量化的监控数据 |
| ⭐ monitoring | 监控 | Prometheus + Grafana 系统健康监控组合 |
| ⭐ namespace | 命名空间（K8s） | `K8s namespace` 在同一集群中隔离不同团队/环境 |
| ⭐ nat | 网络地址转换 | `NAT gateway` 让私有子网中的实例访问公网 |
| ⭐ node | 节点（K8s） | `worker node` 运行 Pod 的物理/虚拟机器 |
| ⭐ observability | 可观测性 | 指标 + 日志 + 链路追踪，系统状态透明化 |
| ⭐ on-call | 值班 | `on-call rotation` 负责响应生产告警的轮值工程师 |
| ⭐ pipeline | 流水线 | `CI/CD pipeline` 自动化构建→测试→部署 |
| ⭐ pod | Pod（K8s） | K8s 最小部署单元，包含一个或多个容器 |
| ⭐ postmortem | 故障复盘 | 分析故障根因和改进措施，`blameless postmortem` 无责追责 |
| ⭐ provision | 开通；配置 | `provision infrastructure` 自动创建云资源 |
| ⭐ pvc | 持久卷申请（K8s） | `PersistentVolumeClaim` 申请持久化存储资源 |
| ⭐ rbac | 基于角色的访问控制 | `K8s RBAC` 控制谁能对哪些资源执行什么操作 |
| rds | 托管数据库（AWS） | `RDS MySQL/PostgreSQL` 全托管，自动备份/扩容 |
| ⭐ region | 区域 | `us-east-1` / `ap-northeast-1` 地理区域 |
| ⭐ registry | 镜像仓库 | `Docker Hub` / `ECR` / `Harbor` 存储和分发镜像 |
| ⭐ replica | 副本数 | `replicas: 3` 运行 3 个 Pod 实例保证高可用 |
| ⭐ rollback | 回滚 | 部署失败时恢复到上一个稳定版本 |
| ⭐ rollout | 滚动发布 | `kubectl rollout status` 逐步替换旧版本 Pod |
| ⭐ route table | 路由表 | 控制子网内流量的下一跳走向 |
| ⭐ runbook | 运维手册 | 标准化的故障处理步骤文档，供 on-call 参考 |
| runner | CI 执行器 | `GitHub Actions runner` 执行流水线任务的机器 |
| ⭐ s3 | 对象存储（AWS） | `S3 bucket` 存文件、静态网站、大数据集 |
| ⭐ secret | 密钥配置（K8s） | `K8s Secret` 加密存储密码/Token 等敏感配置 |
| ⭐ security group | 安全组 | AWS 实例级别的虚拟防火墙，控制入/出站流量 |
| ⭐ serverless | 无服务器 | `AWS Lambda / Vercel` 无需管理服务器的运行模式 |
| service mesh | 服务网格 | `Istio / Linkerd` 管理微服务间的流量、安全、可观测 |
| ⭐ sre | 站点可靠性工程师 | SRE 用软件工程方法保障系统稳定性和可靠性 |
| ⭐ stage | 流水线阶段 | `build → test → deploy` CI/CD 的各执行阶段 |
| ⭐ staging | 预发布环境 | 与生产配置相同，用于上线前最终验证 |
| ⭐ statefulset | 有状态应用控制器（K8s） | 数据库等有状态服务，保证 Pod 身份和存储稳定 |
| ⭐ subnet | 子网 | `private subnet` / `public subnet` VPC 内的网段划分 |
| ⭐ terraform | 基础设施即代码 | `terraform apply` 用代码声明和管理云资源 |
| ⭐ volume | 持久化存储卷 | `mount volume` 容器重启后数据不丢失 |
| ⭐ vpc | 虚拟私有网络 | `AWS VPC` 云上的隔离私有网络环境 |
| waf | Web 应用防火墙 | `WAF rules` 拦截 SQL 注入、XSS 等恶意请求 |
| zero trust | 零信任 | 默认不信任任何请求，每次访问都需验证 |
| ⭐ yaml | YAML 配置格式 | K8s / Docker Compose / GitHub Actions 配置文件格式 |
| ⭐ ddos | 分布式拒绝服务攻击 | 大量机器同时向目标发请求，耗尽资源 |
| dhcp | 动态主机配置协议 | 自动给设备分配 IP 地址 |
| icmp | 网络控制消息协议 | `ping` 命令底层使用 ICMP |
| smtp | 简单邮件传输协议 | `SMTP port 587` 发送邮件的协议 |
| sftp | 安全文件传输协议 | SSH 加密的文件传输，替代 FTP |
| red team | 红队 | 模拟攻击者发现系统漏洞的安全团队 |
| blue team | 蓝队 | 防御和监控安全威胁的运维安全团队 |
| forensics | 数字取证 | 事故后分析日志和证据追溯攻击过程 |

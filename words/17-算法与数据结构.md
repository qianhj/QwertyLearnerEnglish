# 17 算法与数据结构

> 面试高频、LeetCode 常见、计算机基础核心词汇

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ algorithm | 算法 | 解决特定问题的步骤和方法 |
| ⭐ data structure | 数据结构 | 组织和存储数据的方式，如数组、链表、树 |
| ⭐ complexity | 复杂度 | `time complexity` 时间复杂度，`space complexity` 空间复杂度 |
| ⭐ Big O | 大 O 表示法 | `O(n), O(log n), O(n²)` 描述算法性能上界 |
| ⭐ array | 数组 | 连续内存存储的相同类型元素集合 |
| ⭐ linked list | 链表 | 节点通过指针连接，`singly linked list` 单向链表 |
| ⭐ stack | 栈 | LIFO 后进先出，`push` 入栈 `pop` 出栈 |
| ⭐ queue | 队列 | FIFO 先进先出，`enqueue` 入队 `dequeue` 出队 |
| ⭐ priority queue | 优先队列 | 按优先级出队，常用堆实现 |
| ⭐ deque | 双端队列 | 两端都可插入删除的队列 |
| ⭐ hash table | 哈希表 | 键值对存储，`hash function` 哈希函数计算索引 |
| ⭐ hash map | 哈希映射 | `HashMap` 基于哈希表的 Map 接口实现 |
| ⭐ hash set | 哈希集合 | 去重集合，基于哈希表实现 |
| ⭐ collision | 哈希冲突 | 不同键映射到相同索引，用拉链法/开放寻址解决 |
| ⭐ binary tree | 二叉树 | 每个节点最多两个子节点的树结构 |
| ⭐ binary search tree | 二叉搜索树 | BST，左子树 < 根 < 右子树 |
| ⭐ balanced tree | 平衡树 | AVL、红黑树，左右子树高度差受限 |
| ⭐ AVL tree | AVL 树 | 自平衡二叉搜索树，左右高度差 ≤ 1 |
| ⭐ red-black tree | 红黑树 | 自平衡 BST，通过染色和旋转保持平衡 |
| ⭐ B-tree | B 树 | 多路平衡搜索树，数据库和文件系统常用 |
| ⭐ B+ tree | B+ 树 | B 树变种，数据都在叶子节点，适合磁盘存储 |
| ⭐ heap | 堆 | 完全二叉树，父节点 ≥(大顶堆)或≤(小顶堆)子节点 |
| ⭐ min heap | 小顶堆 | 根节点最小，用于找 Top K 小元素 |
| ⭐ max heap | 大顶堆 | 根节点最大，用于找 Top K 大元素 |
| ⭐ trie | 前缀树 / 字典树 | 用于高效存储和检索字符串，如自动补全 |
| ⭐ graph | 图 | 由顶点和边组成的数据结构 |
| ⭐ vertex | 顶点 | 图中的节点，也称 node |
| ⭐ edge | 边 | 连接两个顶点的线，有向/无向 |
| ⭐ directed graph | 有向图 | 边有方向的图 |
| ⭐ undirected graph | 无向图 | 边无方向的图 |
| ⭐ weighted graph | 带权图 | 边有权重的图，如地图距离 |
| ⭐ adjacency list | 邻接表 | 图的存储方式，每个顶点存邻居列表 |
| ⭐ adjacency matrix | 邻接矩阵 | 图的存储方式，二维数组表示边 |
| ⭐ DFS | 深度优先搜索 | Depth-First Search，用栈或递归实现 |
| ⭐ BFS | 广度优先搜索 | Breadth-First Search，用队列实现 |
| ⭐ topological sort | 拓扑排序 | 对有向无环图进行线性排序 |
| ⭐ shortest path | 最短路径 | Dijkstra、Floyd-Warshall 算法求解 |
| ⭐ Dijkstra | 迪杰斯特拉算法 | 单源最短路径，贪心策略 |
| ⭐ minimum spanning tree | 最小生成树 | MST，`Prim` 和 `Kruskal` 算法 |
| ⭐ union find | 并查集 | 处理不相交集合的合并与查询 |
| ⭐ sorting | 排序 | 将元素按特定顺序排列 |
| ⭐ quick sort | 快速排序 | 分治，选 pivot 分区，平均 O(n log n) |
| ⭐ merge sort | 归并排序 | 分治，合并两个有序数组，稳定 O(n log n) |
| ⭐ heap sort | 堆排序 | 利用堆进行排序，O(n log n) |
| ⭐ bubble sort | 冒泡排序 | 相邻元素比较交换，O(n²) |
| ⭐ insertion sort | 插入排序 | 逐个插入到正确位置，小数据量高效 |
| ⭐ selection sort | 选择排序 | 每次选最小元素，O(n²) |
| ⭐ counting sort | 计数排序 | 非比较排序，统计元素出现次数 |
| ⭐ radix sort | 基数排序 | 按位排序，O(nk) k 为数字位数 |
| ⭐ stable sort | 稳定排序 | 相等元素保持原有相对顺序 |
| ⭐ in-place | 原地排序 | 不使用额外空间或 O(1) 额外空间 |
| ⭐ recursion | 递归 | 函数调用自身，需基准条件和递归条件 |
| ⭐ iteration | 迭代 | 用循环重复执行代码块 |
| ⭐ backtracking | 回溯 | 尝试所有可能，不行就回退，如八皇后 |
| ⭐ divide and conquer | 分治法 | 大问题分解为小问题，分别解决再合并 |
| ⭐ dynamic programming | 动态规划 | DP，最优子结构、重叠子问题、状态转移 |
| ⭐ memoization | 记忆化 | 缓存子问题结果避免重复计算 |
| ⭐ tabulation | 表格法 | 自底向上填表实现 DP |
| ⭐ greedy | 贪心算法 | 每一步选局部最优，期望全局最优 |
| ⭐ two pointers | 双指针 | 左右指针或快慢指针遍历数组 |
| ⭐ sliding window | 滑动窗口 | 维护一个区间处理子数组问题 |
| ⭐ binary search | 二分查找 | 有序数组中 O(log n) 查找 |
| ⭐ bit manipulation | 位运算 | 用 & \| ^ ~ << >> 操作二进制位 |
| ⭐ time limit exceeded | 超时 | TLE，算法时间复杂度不够优化 |
| ⭐ memory limit exceeded | 超内存 | MLE，空间复杂度太高 |
| ⭐ depth | 深度 | 树中节点到根的距离 |
| ⭐ height | 高度 | 树中节点到叶子的最远距离 |
| ⭐ leaf node | 叶子节点 | 没有子节点的节点 |
| ⭐ root node | 根节点 | 树的顶端节点，没有父节点 |
| ⭐ parent | 父节点 | 直接上级节点 |
| ⭐ child | 子节点 | 直接下级节点 |
| ⭐ sibling | 兄弟节点 | 同一父节点的节点 |
| ⭐ ancestor | 祖先节点 | 从根到该节点路径上的所有节点 |
| ⭐ descendant | 后代节点 | 该节点子树中的所有节点 |
| ⭐ subtree | 子树 | 以某节点为根的树 |
| ⭐ degree | 度 | 节点拥有的子节点数 |
| ⭐ path | 路径 | 顶点序列，相邻顶点间有边 |
| ⭐ cycle | 环 | 起点终点相同的路径 |
| ⭐ acyclic | 无环的 | `DAG` 有向无环图 Directed Acyclic Graph |
| ⭐ connectivity | 连通性 | 图中节点间是否可达 |
| ⭐ strongly connected | 强连通 | 有向图中任意两点互相可达 |
| ⭐ clique | 团 | 子图中任意两点都有边相连 |
| ⭐ bipartite | 二分图 | 顶点可分为两组，组内无边 |
| ⭐ brute force | 暴力枚举 | 尝试所有可能解法 |
| ⭐ heuristic | 启发式 | 经验法则，不一定最优但快速 |
| ⭐ amortized analysis | 摊还分析 | 一系列操作的平均时间分析 |

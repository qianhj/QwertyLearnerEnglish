# 04 后端 & 服务

> 后端开发、API 设计、服务架构、异步编程、安全相关词汇

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ ack / nack | 确认 / 拒绝消息 | `message ack` 消费者告知已处理；`nack` 处理失败退回 |
| ⭐ async / await | 异步 / 等待 | `async function fetchData()` 非阻塞执行 |
| ⭐ authentication | 认证 | `JWT authentication` JWT 认证 |
| ⭐ authorization | 授权 | `role-based authorization` 基于角色授权 |
| ⭐ backoff | 退避 | `exponential backoff` 指数级延迟重试，避免雪崩 |
| ⭐ backpressure | 背压 | 上游速率超过下游时的流量控制机制 |
| ⭐ broker | 消息代理 | `Kafka broker` 消息路由和存储节点 |
| ⭐ cache | 缓存 | `Redis cache` Redis 缓存 |
| ⭐ callback | 回调函数 | `callback hell` 嵌套回调问题 |
| ⭐ circuit breaker | 熔断器 | 服务故障时快速失败，防止级联崩溃 |
| ⭐ consumer | 消费者 | `consumer group` 消费消息队列的服务 |
| ⭐ controller | 控制器 | `MVC controller` 处理 HTTP 请求的控制器 |
| ⭐ cors | 跨域资源共享 | `CORS policy` 跨域策略，允许指定来源 |
| ⭐ cqrs | 命令查询职责分离 | 读写分离的架构模式 |
| ⭐ csrf | 跨站请求伪造 | `CSRF token` 表单令牌防伪造攻击 |
| ⭐ dashboard | 仪表板 | `Grafana dashboard` 实时展示系统指标 |
| ⭐ dead letter | 死信队列 | 无法处理的消息存入 `dead letter queue` |
| ⭐ dependency injection | 依赖注入 | `DI container` 解耦对象创建和使用 |
| ⭐ deprecate | 废弃 | `deprecated API` 即将下线的接口，文档会标注 |
| ⭐ endpoint | 接口端点 | `API endpoint /users` |
| ⭐ event-driven | 事件驱动 | `event-driven architecture` 解耦异步处理 |
| ⭐ event loop | 事件循环 | Node.js 单线程模型的核心机制 |
| ⭐ factory | 工厂模式 | `factory pattern` 统一封装对象创建逻辑 |
| ⭐ fallback | 降级 | `fallback response` 服务不可用时的兜底返回 |
| ⭐ fanout | 广播 | `fanout exchange` 向所有绑定队列广播消息 |
| ⭐ gateway | 网关 | `API gateway` 统一入口，处理路由/鉴权/限流 |
| graphql | GraphQL | `GraphQL query` 客户端按需获取字段，减少过度获取 |
| grpc | gRPC 远程调用 | `gRPC service` 高性能二进制协议，适合微服务 |
| ⭐ harden | 安全加固 | `security hardening` 减少攻击面 |
| ⭐ header | 请求/响应头 | `Authorization header` / `Content-Type header` |
| ⭐ idempotent | 幂等的 | `idempotent request` 相同请求多次执行结果相同 |
| ⭐ jwt | JSON Web Token | `Bearer <token>` 无状态令牌认证 |
| ⭐ latency | 延迟 | `p99 latency` 第 99 百分位响应时间 |
| ⭐ load balancer | 负载均衡器 | 分发流量到多台服务器，避免单点压力 |
| ⭐ message queue | 消息队列 | `MQ` 异步通信缓冲，解耦生产者和消费者 |
| ⭐ microservice | 微服务 | `microservice architecture` 按业务拆分的独立服务 |
| ⭐ middleware | 中间件 | `Express middleware` 请求处理管道中的插件 |
| ⭐ migration | 数据库迁移 | `run migrations` 执行表结构变更脚本 |
| ⭐ model | 数据模型 | `ORM model` 对象关系映射的数据结构定义 |
| monolith | 单体应用 | `monolithic architecture` 所有功能在一个进程中 |
| ⭐ oauth | OAuth 授权框架 | `OAuth 2.0` 第三方登录（"用 Google 登录"） |
| offset | 偏移量 | `Kafka offset` 标记消费者读到队列的哪个位置 |
| openapi / swagger | 接口文档规范 | `Swagger UI` 可视化 API 文档和测试界面 |
| ⭐ orm | 对象关系映射 | `Prisma / Sequelize ORM` 用代码操作数据库 |
| ⭐ pagination | 分页 | `cursor-based pagination` 游标分页性能更好 |
| ⭐ payload | 载荷；请求体 | `request payload` 请求携带的数据 |
| ⭐ producer | 生产者 | 发送消息到队列的一方 |
| ⭐ profiler | 性能分析器 | `CPU profiler` 找出程序中的性能瓶颈 |
| ⭐ promise | Promise 对象 | `Promise.then().catch()` 异步操作的封装 |
| ⭐ proxy | 代理 | `reverse proxy` Nginx 反向代理，转发请求 |
| pub/sub | 发布/订阅模式 | `pub/sub` 解耦发布者与订阅者 |
| ⭐ queue | 队列 | `task queue` 先进先出的任务缓冲 |
| ⭐ rate limit | 速率限制 | `429 Too Many Requests` 超出调用频率 |
| ⭐ repository | 仓储模式 | `UserRepository` 封装数据库访问层 |
| ⭐ rest | REST 风格 | `RESTful API` 基于 HTTP 方法的接口规范 |
| ⭐ retry | 重试 | `retry with backoff` 失败后延迟重试 |
| ⭐ route | 路由 | `POST /api/users` 路由定义 |
| ⭐ saga | Saga 模式 | 分布式事务的编排方案，通过事件链实现 |
| ⭐ sanitize | 净化输入 | `sanitize HTML` 过滤用户输入防注入攻击 |
| ⭐ schema | 数据结构定义 | `JSON schema` / `DB schema` |
| ⭐ serialize | 序列化 | `serialize response` 对象转为 JSON/二进制 |
| ⭐ service | 业务服务层 | `UserService` 封装业务逻辑的服务类 |
| ⭐ singleton | 单例模式 | `singleton instance` 全局唯一实例 |
| ⭐ sla | 服务等级协议 | `SLA 99.9%` 对可用性的承诺（每月宕机 < 43 分钟） |
| ⭐ socket | 套接字 | `WebSocket` 双向实时通信 |
| ⭐ span | 追踪单元 | `trace span` 分布式追踪中一次调用的时间段 |
| soap | SOAP 协议 | 企业级 XML 接口协议，常见于老系统集成 |
| ⭐ sso | 单点登录 | `SSO` 一次登录，访问多个系统 |
| ⭐ ssl / tls | 加密传输协议 | `HTTPS = HTTP + TLS` |
| ⭐ stateless | 无状态的 | `stateless REST API` 服务端不保存会话状态 |
| ⭐ throttle | 节流 | `throttle requests` 限制接口调用频率 |
| throughput | 吞吐量 | `requests per second` 每秒处理的请求数 |
| ⭐ timeout | 超时 | `request timeout 5000ms` 超时则报错 |
| ⭐ topic | 主题 | `Kafka topic` 消息分类的频道 |
| ⭐ trace / tracing | 链路追踪 | `distributed tracing` Jaeger/Zipkin 追踪跨服务调用 |
| ⭐ ttl | 存活时间 | `TTL 3600s` 缓存/Token 的过期时间 |
| ⭐ uptime | 正常运行时间 | `99.9% uptime` 衡量服务可用性的指标 |
| versioning | 版本管理 | `API versioning /v1/ /v2/` 接口向后兼容 |
| ⭐ webhook | 网络钩子 | `GitHub webhook` 事件触发时主动推送通知 |
| ⭐ xss | 跨站脚本攻击 | `XSS sanitize` 过滤用户输入防恶意脚本注入 |
| ⭐ audit | 审计；审查 | `security audit` 安全审查，记录谁在何时做了什么 |
| ⭐ exploit | 漏洞利用 | 攻击者利用漏洞执行恶意代码 |
| ⭐ vulnerability | 漏洞 | `security vulnerability` 可被攻击者利用的软件缺陷 |
| honeypot | 蜜罐 | 伪装成真实系统吸引攻击者的陷阱 |
| devsecops | DevSecOps | 将安全融入 CI/CD 流程，安全左移 |
| pentest | 渗透测试 | 模拟黑客攻击测试系统安全性 |

# 10 英语语法与常用句式（120）

> 技术文档、GitHub Issue、代码注释中反复出现的语法结构和实用句型

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ make sure (that) | 确保；务必 | `Make sure you have Node.js installed` |
| ⭐ note that | 注意；请注意 | `Note that this is case-sensitive` 注意区分大小写 |
| ⭐ keep in mind | 牢记；注意 | `Keep in mind that X may change` |
| ⭐ be aware (that/of) | 注意；意识到 | `Be aware of potential side effects` |
| ⭐ in order to | 为了；目的是 | `In order to use this, you need to...` |
| ⭐ so that | 以便；使得 | `refactored so that it's reusable` |
| ⭐ as long as | 只要；前提是 | `works as long as the token is valid` |
| ⭐ as well as | 以及；还有 | `supports JSON as well as XML` |
| ⭐ rather than | 而不是；优于 | `use async/await rather than callbacks` |
| ⭐ instead of | 代替；而不是 | `use Map instead of plain object` |
| ⭐ due to | 由于；因为 | `failed due to network timeout` |
| ⭐ as a result (of) | 结果；因此 | `as a result, the request was rejected` |
| ⭐ therefore | 因此；所以 | `the value is null, therefore it crashes` |
| ⭐ however | 然而；但是 | `works on Mac, however fails on Windows` |
| ⭐ whereas | 然而；而（对比） | `Python is dynamic, whereas TypeScript is typed` |
| ⭐ although / though | 尽管；虽然 | `although it works, it's not ideal` |
| ⭐ regardless of | 不管；无论 | `works regardless of OS` 跨平台 |
| ⭐ in addition (to) | 除此之外；另外 | `In addition, you can also...` |
| ⭐ furthermore | 此外；更进一步 | `Furthermore, the new API supports...` |
| ⭐ on the other hand | 另一方面 | 对比两种方案时常用 |
| ⭐ in contrast | 相比之下；与之相反 | `In contrast, the old version...` |
| ⭐ for example / for instance | 例如 | `for example, you can pass a callback` |
| ⭐ such as | 例如；比如 | `languages such as Python, Go, Rust` |
| ⭐ i.e. | 即；也就是说 | 拉丁语缩写，= "that is" |
| ⭐ e.g. | 例如 | 拉丁语缩写，= "for example" |
| ⭐ etc. | 等等 | `...and etc.` 列举未完时使用 |
| ⭐ refer to | 参考；指的是 | `refer to the docs` 参考文档 |
| ⭐ according to | 根据；按照 | `according to the spec` 按照规范 |
| ⭐ based on | 基于；根据 | `based on your config` 根据你的配置 |
| ⭐ given (that) | 鉴于；考虑到 | `given that X is true, Y follows` |
| ⭐ assuming (that) | 假设；如果 | `assuming the token is valid` |
| ⭐ in case (of) | 以防；如果 | `in case of error, fallback to default` |
| ⭐ otherwise | 否则；不然 | `handle the error, otherwise it will crash` |
| ⭐ by default | 默认情况下 | `by default, this is disabled` |
| ⭐ out of the box | 开箱即用 | `works out of the box` 无需额外配置 |
| ⭐ under the hood | 底层；内部实现 | `under the hood, it uses WebSockets` |
| ⭐ on behalf of | 代表；替 | `acting on behalf of the user` |
| ⭐ in terms of | 就……而言；关于 | `in terms of performance` 就性能而言 |
| ⭐ with respect to | 关于；就……来说 | `issues with respect to security` |
| ⭐ as opposed to | 而不是；与……相对 | `declarative, as opposed to imperative` |
| ⭐ is used to | 用于；被用来 | `this method is used to parse JSON` |
| ⭐ is designed to | 旨在；设计用于 | `React is designed to build UIs` |
| ⭐ is intended to | 意图是；目的是 | `this flag is intended to disable X` |
| ⭐ is responsible for | 负责；承担 | `this module is responsible for routing` |
| ⭐ is capable of | 能够；可以 | `is capable of handling 10k requests/s` |
| ⭐ is known as | 被称为 | `this pattern is known as singleton` |
| ⭐ is referred to as | 被称作；也叫做 | `commonly referred to as "type narrowing"` |
| ⭐ failed to | 未能；失败了 | `failed to connect to database` |
| ⭐ unable to | 无法；不能 | `unable to resolve module` |
| ⭐ could not | 无法 | `could not find file 'index.js'` |
| ⭐ expected X but got Y | 期望 X 但得到 Y | 常见测试失败描述 |
| ⭐ X is not defined | X 未定义 | `ReferenceError: foo is not defined` |
| ⭐ X is not a function | X 不是函数 | `TypeError: X is not a function` |
| ⭐ X has no property Y | X 没有属性 Y | `Cannot read property 'y' of undefined` |
| ⭐ it seems (that) | 似乎；看来 | `It seems that the issue is in line 42` |
| ⭐ it turns out (that) | 结果发现；原来 | `It turns out the bug was a typo` |
| ⭐ I noticed that | 我注意到 | GitHub issue 描述问题时常用 |
| ⭐ I ran into | 我遇到了 | `I ran into this error when...` |
| ⭐ I'm getting | 我收到了（错误） | `I'm getting a 403 error` |
| ⭐ have you tried | 你试过……吗 | `Have you tried clearing the cache?` |
| ⭐ would it be possible to | 是否可以 | 礼貌请求功能或帮助 |
| ⭐ feel free to | 随时；请随意 | `feel free to open an issue` |
| ⭐ let me know if | 如果……请告诉我 | `Let me know if this helps` |
| ⭐ as mentioned | 如前所述 | `as mentioned above` 如上所述 |
| ⭐ as shown | 如图所示；如所示 | `as shown in the example` 如示例所示 |
| ⭐ for more details | 更多详情 | `for more details, see the docs` |
| ⭐ see also | 另见；参考 | `See also: related issues` |
| ⭐ to sum up | 总结来说 | `To sum up, avoid using X for Y` |
| ⭐ in summary | 总结 | `In summary, the fix involves...` |
| ⭐ that said | 话虽如此 | `That said, it works for most cases` |
| ⭐ to be fair | 公平地说 | `To be fair, this is an edge case` |
| ⭐ strictly speaking | 严格来说 | `Strictly speaking, this is undefined behavior` |
| ⭐ in practice | 实际上；实践中 | `In practice, you rarely need this` |
| ⭐ in theory | 理论上 | `In theory it should work, but...` |
| ⭐ for the sake of | 为了……起见 | `for the sake of simplicity` 为简单起见 |
| ⭐ at the expense of | 以……为代价 | `fast, at the expense of memory` |
| ⭐ trade-off | 权衡；取舍 | `there's a trade-off between speed and memory` |
| ⭐ pros and cons | 优缺点 | `the pros and cons of this approach` |
| ⭐ as a rule of thumb | 根据经验；一般来说 | `As a rule of thumb, avoid premature optimization` |
| ⭐ best practice | 最佳实践 | `follow best practices` |
| ⭐ common pitfall | 常见陷阱 | `a common pitfall is forgetting to await` |
| ⭐ gotcha | 坑；容易踩的陷阱 | `this is a common gotcha in JS` |
| ⭐ caveat | 警告；注意事项 | `one caveat: this only works in Chrome` |
| ⭐ disclaimer | 免责声明；说明 | `disclaimer: this is not production-ready` |
| ⭐ heads up | 提前告知；注意 | `heads up: this API will be removed in v3` |
| ⭐ breaking | 破坏性的 | `⚠️ breaking change in this version` |
| ⭐ wip | 进行中（Work In Progress） | `WIP: do not merge` 草稿勿合并 |
| ⭐ lgtm | 看起来不错（Looks Good To Me） | 代码审查通过时常用 |
| ⭐ tldr | 太长不看（Too Long Didn't Read） | `TL;DR: just use option A` |
| ⭐ fyi | 供参考（For Your Information） | `FYI, this was fixed in v2.1` |
| ⭐ afaik | 据我所知（As Far As I Know） | `AFAIK, this is the intended behavior` |
| ⭐ iirc | 如果我没记错（If I Recall Correctly） | `IIRC, this was changed in v1.5` |
| ⭐ imho | 在我看来（In My Humble Opinion） | `IMHO, the API design could be better` |

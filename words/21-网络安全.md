# 21 网络安全

> 渗透测试、OWASP、漏洞攻防、安全防护相关词汇

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ cybersecurity | 网络安全 | 保护系统和数据免受攻击的领域 |
| ⭐ penetration testing / pentest | 渗透测试 | 模拟攻击者寻找系统漏洞 |
| ⭐ vulnerability | 漏洞 | 系统可被利用的弱点 |
| ⭐ exploit | 漏洞利用 | 利用漏洞的攻击代码或技术 |
| ⭐ POC | 概念验证 | Proof of Concept，验证漏洞存在的代码 |
| ⭐ payload | 有效载荷 | 攻击中实际执行的恶意代码 |
| ⭐ shellcode | Shell 代码 | 打开命令行会话的机器码 |
| ⭐ CVE | 漏洞编号 | Common Vulnerabilities and Exposures |
| ⭐ CVSS | 漏洞评分 | Common Vulnerability Scoring System，0-10分 |
| ⭐ zero day | 零日漏洞 | 未公开、无补丁的新漏洞 |
| ⭐ OWASP | 开源Web安全项目 | Open Web Application Security Project |
| ⭐ OWASP Top 10 | 十大安全风险 | Web应用最常见漏洞清单 |
| ⭐ SQL injection / SQLi | SQL注入 | 通过输入注入恶意SQL语句 |
| ⭐ XSS | 跨站脚本攻击 | Cross-Site Scripting，注入恶意脚本 |
| ⭐ CSRF | 跨站请求伪造 | Cross-Site Request Forgery |
| ⭐ SSRF | 服务端请求伪造 | Server-Side Request Forgery |
| ⭐ RCE | 远程代码执行 | Remote Code Execution，最严重漏洞类型 |
| ⭐ LFI / RFI | 本地/远程文件包含 | Local/Remote File Inclusion |
| ⭐ XXE | XML外部实体攻击 | XML External Entity |
| ⭐ IDOR | 不安全直接对象引用 | Insecure Direct Object Reference |
| ⭐ MITM | 中间人攻击 | Man-in-the-Middle，窃听或篡改通信 |
| ⭐ phishing | 钓鱼攻击 | 伪装可信来源骗取信息 |
| ⭐ malware | 恶意软件 | 病毒、木马、勒索软件等 |
| ⭐ ransomware | 勒索软件 | 加密文件索要赎金的恶意软件 |
| ⭐ trojan | 木马 | 伪装成正常程序的恶意软件 |
| ⭐ virus | 病毒 | 自我复制破坏系统的恶意代码 |
| ⭐ worm | 蠕虫 | 通过网络自我传播的程序 |
| ⭐ backdoor | 后门 | 绕过正常认证的隐藏入口 |
| ⭐ rootkit |  rootkit | 隐藏自身和恶意程序的工具集 |
| ⭐ botnet | 僵尸网络 | 被控制的受感染设备网络 |
| ⭐ DDoS | 分布式拒绝服务 | 大量请求使服务瘫痪 |
| ⭐ firewall | 防火墙 | 监控和控制网络流量的安全设备 |
| ⭐ WAF | Web应用防火墙 | Web Application Firewall |
| ⭐ IDS | 入侵检测系统 | Intrusion Detection System |
| ⭐ IPS | 入侵防御系统 | Intrusion Prevention System |
| ⭐ SIEM | 安全信息和事件管理 | Security Information and Event Management |
| ⭐ VPN | 虚拟专用网络 | Virtual Private Network，加密通信隧道 |
| ⭐ TLS / SSL | 传输层安全协议 | HTTPS 的基础加密协议 |
| ⭐ certificate | 证书 | 用于身份验证和加密的数字证书 |
| ⭐ CA | 证书颁发机构 | Certificate Authority，如 Let's Encrypt |
| ⭐ encryption | 加密 | 将数据转换为不可读格式 |
| ⭐ decryption | 解密 | 将加密数据还原 |
| ⭐ symmetric | 对称加密 | 加密解密用同一密钥，如 AES |
| ⭐ asymmetric | 非对称加密 | 公钥加密私钥解密，如 RSA |
| ⭐ hash | 哈希 | 单向函数，如 SHA-256 |
| ⭐ salt | 盐值 | 随机数据增加哈希安全性 |
| ⭐ brute force | 暴力破解 | 尝试所有可能的密码组合 |
| ⭐ dictionary attack | 字典攻击 | 用常见密码列表尝试破解 |
| ⭐ credential stuffing | 凭据填充 | 用泄露的用户名密码批量尝试 |
| ⭐ session hijacking | 会话劫持 | 窃取用户会话令牌 |
| ⭐ cookie theft | Cookie窃取 | 盗取浏览器Cookie冒充用户 |
| ⭐ token | 令牌 | 身份验证的凭证 |
| ⭐ JWT | JSON Web Token | 常用的令牌格式 |
| ⭐ OAuth | 开放授权协议 | 第三方登录授权标准 |
| ⭐ MFA / 2FA | 多因素认证 | Multi-Factor / Two-Factor Authentication |
| ⭐ brute force | 暴力破解 | 穷举所有可能的密钥或密码 |
| ⭐ port scanning | 端口扫描 | 探测目标开放的端口 |
| ⭐ network sniffing | 网络嗅探 | 捕获和分析网络流量 |
| ⭐ packet | 数据包 | 网络传输的基本单位 |
| ⭐ TCP / UDP | 传输协议 | Transmission Control / User Datagram Protocol |
| ⭐ IP | 互联网协议 | Internet Protocol |
| ⭐ DNS | 域名系统 | Domain Name System |
| ⭐ DNS hijacking | DNS劫持 | 篡改DNS解析结果 |
| ⭐ ARP spoofing | ARP欺骗 | 伪造ARP响应进行中间人攻击 |
| ⭐ sandbox | 沙箱 | 隔离的执行环境 |
| ⭐ honeypot | 蜜罐 | 诱捕攻击者的伪装系统 |
| ⭐ blue team | 蓝队 | 防守方安全团队 |
| ⭐ red team | 红队 | 攻击方模拟团队 |
| ⭐ purple team | 紫队 | 红蓝协作的安全团队 |
| ⭐ bug bounty | 漏洞赏金 | 奖励发现安全漏洞的研究人员 |
| ⭐ responsible disclosure | 负责任的披露 | 向厂商报告漏洞并给予修复时间 |
| ⭐ hardening | 加固 | 减少系统攻击面的安全措施 |
| ⭐ patching | 打补丁 | 安装安全更新修复漏洞 |
| ⭐ audit | 审计 | 检查系统和代码的安全性 |
| ⭐ compliance | 合规 | 符合安全标准和法规 |
| ⭐ GDPR | 欧盟数据保护条例 | General Data Protection Regulation |
| ⭐ HIPAA | 健康信息保护法案 | 美国医疗数据保护 |
| ⭐ SOC 2 | 服务组织控制报告 | 企业安全合规认证 |
| ⭐ ISO 27001 | 信息安全标准 | 国际信息安全管理体系标准 |
| ⭐ information gathering | 信息收集 | 侦察阶段收集目标信息 |
| ⭐ reconnaissance | 侦察 | 渗透测试前期信息搜集 |
| ⭐ enumeration | 枚举 | 系统地收集系统信息 |
| ⭐ fuzzing | 模糊测试 | 输入随机数据发现崩溃 |
| ⭐ reverse engineering | 逆向工程 | 分析二进制程序理解其逻辑 |
| ⭐ social engineering | 社会工程学 | 利用人性弱点获取信息 |

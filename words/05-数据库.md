# 05 数据库

> SQL、NoSQL、事务、锁、复制、性能优化相关词汇

| 英文单词 / 短语 | 中文释义 | 场景记忆 |
|---|---|---|
| ⭐ acid | ACID 特性 | 原子性+一致性+隔离性+持久性，事务的四个保证 |
| ⭐ aggregate | 聚合 | `GROUP BY` / `aggregate function` COUNT/SUM/AVG |
| alias | 别名 | `SELECT u.name AS username` 用 AS 起别名 |
| ⭐ atomicity | 原子性 | 事务要么全部成功，要么全部回滚 |
| avg | 平均值函数 | `SELECT AVG(price) FROM orders` |
| ⭐ binlog | 二进制日志 | MySQL `binlog` 用于主从复制和数据恢复 |
| ⭐ cascade | 级联 | `ON DELETE CASCADE` 删除主记录同时删除关联记录 |
| ⭐ collection | 集合（NoSQL） | MongoDB 中对应 SQL 的表 |
| ⭐ column | 列；字段 | `add column email VARCHAR(255)` |
| ⭐ commit | 提交事务 | `COMMIT` 将事务的变更永久保存 |
| ⭐ constraint | 约束 | `UNIQUE constraint` / `NOT NULL constraint` |
| count | 计数函数 | `SELECT COUNT(*) FROM users` |
| ⭐ cursor | 游标 | `cursor-based pagination` 高性能分页方案 |
| ⭐ deadlock | 死锁 | `deadlock detected` 两个事务互相等待对方释放锁 |
| ⭐ delete | 删除 | `DELETE FROM table WHERE id=1` |
| ⭐ distinct | 去重 | `SELECT DISTINCT country` 去除重复值 |
| ⭐ document | 文档（NoSQL） | MongoDB `document` 存储 JSON 格式的记录 |
| embedding | 向量嵌入 | AI 语义搜索中存储的高维向量数据 |
| ⭐ explain | 执行计划 | `EXPLAIN SELECT` 查看查询是否命中索引 |
| ⭐ foreign key | 外键 | `FOREIGN KEY (user_id) REFERENCES users(id)` |
| full scan | 全表扫描 | 未命中索引时逐行扫描，数据量大时极慢 |
| ⭐ group by | 分组 | `GROUP BY user_id` 按字段分组后聚合 |
| ⭐ having | 分组后过滤 | `HAVING COUNT(*) > 5` 对聚合结果过滤 |
| ⭐ index | 索引 | `CREATE INDEX` 加速查询，空间换时间 |
| ⭐ insert | 插入 | `INSERT INTO users (name) VALUES ('Alice')` |
| ⭐ isolation level | 隔离级别 | `READ COMMITTED` 已提交读，控制并发可见性 |
| ⭐ join | 关联查询 | `INNER JOIN` / `LEFT JOIN` 连接多表 |
| ⭐ key-value | 键值存储 | Redis `key-value store` 最简单的 NoSQL 结构 |
| like | 模糊匹配 | `WHERE name LIKE 'John%'` % 匹配任意字符 |
| ⭐ lock | 锁 | `row-level lock` 行级锁，控制并发写入 |
| ⭐ materialized view | 物化视图 | 预计算并持久化的查询结果，查询更快 |
| max | 最大值函数 | `SELECT MAX(score) FROM results` |
| ⭐ migration | 迁移脚本 | `schema migration` 数据库结构变更脚本 |
| min | 最小值函数 | `SELECT MIN(price) FROM products` |
| ⭐ mvcc | 多版本并发控制 | 读不阻塞写的并发方案，PostgreSQL/InnoDB 使用 |
| ⭐ normalize | 规范化 | `database normalization` 减少数据冗余 |
| ⭐ nosql | 非关系型数据库 | MongoDB / Redis / Cassandra，灵活结构 |
| ⭐ null | 空值 | `IS NULL` / `NOT NULL constraint` |
| ⭐ optimistic lock | 乐观锁 | 假设不冲突，提交时用版本号校验 |
| ⭐ order by | 排序 | `ORDER BY created_at DESC` 按时间降序 |
| ⭐ partition | 分区 | `table partitioning` 按范围/哈希水平切分大表 |
| ⭐ pessimistic lock | 悲观锁 | 操作前先加锁，阻止其他事务并发修改 |
| ⭐ primary key | 主键 | `PRIMARY KEY (id)` 唯一标识每条记录 |
| ⭐ procedure | 存储过程 | `CALL stored_procedure()` 数据库端的可执行程序 |
| ⭐ query | 查询 | `SQL query` / `query builder` |
| ⭐ record | 记录；行 | `insert record` 插入一条数据 |
| ⭐ relation | 关系 | `one-to-many relation` 一对多关系 |
| ⭐ relational / rdbms | 关系型数据库 | MySQL / PostgreSQL / Oracle |
| ⭐ replica | 副本 | `read replica` 只读副本，分担主库查询压力 |
| ⭐ replication | 复制 | `master-slave replication` 主从数据同步 |
| replication lag | 复制延迟 | 主从同步的时间差，影响读一致性 |
| ⭐ rollback | 回滚 | `ROLLBACK` 撤销未提交的事务变更 |
| ⭐ row | 行 | `row count` 表中的数据行数 |
| ⭐ savepoint | 保存点 | `SAVEPOINT sp1` 事务内部的局部回滚点 |
| ⭐ schema | 数据库结构 | `database schema` 表结构和关系的定义 |
| ⭐ select | 查询 | `SELECT * FROM users WHERE active=1` |
| ⭐ shard | 分片 | `database sharding` 水平拆分到多个数据库实例 |
| snapshot | 快照 | `point-in-time snapshot` 某时刻的数据状态备份 |
| stored procedure | 存储过程 | 封装在数据库中的可重用 SQL 程序 |
| subquery | 子查询 | `SELECT * FROM (SELECT ...) AS sub` 嵌套查询 |
| sum | 求和函数 | `SELECT SUM(amount) FROM orders` |
| ⭐ table | 表 | `CREATE TABLE users (...)` |
| time-series | 时序数据库 | InfluxDB 存储按时间顺序的监控/日志指标 |
| ⭐ transaction | 事务 | `BEGIN ... COMMIT / ROLLBACK` 原子操作单元 |
| ⭐ trigger | 触发器 | `BEFORE INSERT trigger` 数据变更时自动执行 |
| ⭐ ttl | 数据过期时间 | Redis/MongoDB `TTL` 自动删除过期数据 |
| ⭐ unique | 唯一约束 | `UNIQUE (email)` 该字段不允许重复值 |
| ⭐ union | 联合查询 | `UNION ALL` 合并多个 SELECT 结果集 |
| ⭐ update | 更新 | `UPDATE users SET name='Bob' WHERE id=1` |
| vacuum | 清理（PG） | PostgreSQL `VACUUM` 回收死行，释放空间 |
| vector db | 向量数据库 | Pinecone/Weaviate，存储 AI 语义搜索的向量 |
| ⭐ view | 视图 | `CREATE VIEW` 基于查询创建的虚拟表 |
| ⭐ wal | 预写日志 | Write-Ahead Log，数据库崩溃恢复的基础 |
| ⭐ where | WHERE 条件 | `WHERE id > 100 AND status='active'` |
| compaction | 压缩整理 | LSM-Tree 合并小文件，RocksDB/Cassandra 定期执行 |
| tombstone | 墓碑标记 | 软删除时留下的标记，后续 compaction 时清理 |
| ⭐ mysql | MySQL | 最流行的开源关系型数据库，`mysql -u root -p` |
| ⭐ postgres | PostgreSQL | 功能最强大的开源关系型数据库，支持 JSON/全文搜索 |
| ⭐ mongodb | MongoDB | 最流行的文档型 NoSQL 数据库，存储 BSON |
| ⭐ redis | Redis | 内存键值数据库，常用于缓存/会话/消息队列 |
| memcached | Memcached | 简单高速的分布式内存缓存，比 Redis 功能少 |
| oracle | Oracle DB | 企业级关系型数据库，金融/电信常见 |
